# 文档同步规则

## 触发时机

每次完成代码修改（重构阶段完成后），自动执行以下检查。

## 检查清单

### 1. CLAUDE.md
命中任意一条就更新：
- 新增了模块/文件 → 更新目录结构部分
- 新增了构建/测试命令 → 更新常用命令部分
- 修改了核心原则或约定 → 更新对应内容
- **可以直接改，改完告知开发者**

### 2. CHANGELOG.md
每个功能实现完毕后，追加一条记录：
```
## [Unreleased]
### Added
- feat(模块名): 简要描述
```
- **可以直接改**

### 3. docs/design.md
- 对比实际实现与文档的差异（接口签名、返回类型、API 路径等）
- **禁止擅自修改**
- 发现差异时，报告给开发者，让开发者选择：
  A) 按文档修正代码
  B) 更新文档以反映实际实现

### 4. README.md（如果存在）
- 新增对外 API → 更新使用示例
- 依赖变更 → 更新安装说明
- **可以直接改，改完告知开发者**

## 输出格式

每次检查后输出：
```
📄 文档同步检查
├── CLAUDE.md      ✅ 无需更新 / ⚠️ 已更新：[改了什么]
├── CHANGELOG.md   ✅ 已追加 / ⚠️ 已追加：[内容]
├── docs/design.md ✅ 一致 / ⚠️ 有偏差：[差异，等你决定 A/B]
└── README.md      ✅ 无需更新 / ⚠️ 已更新：[改了什么]
```
